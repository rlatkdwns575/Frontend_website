# 밸런스 게임 웹사이트

재미있는 밸런스 게임 질문에 투표하고, 각 선택지의 결과를 바로 확인할 수 있는 간단한 웹 프로젝트입니다.

이 프로젝트는 HTML, CSS, JavaScript만으로 구현되었으며, 별도의 서버 없이 브라우저에서 바로 실행할 수 있습니다. 또한 `localStorage`를 사용해 같은 브라우저에서는 이전 투표 결과가 유지됩니다.

## 주요 기능

- 35개의 밸런스 게임 질문 제공
- 두 선택지 중 하나를 클릭해 투표 가능
- 투표 직후 각 선택지의 퍼센트와 표 수 표시
- 한 질문에서 중복 투표 방지
- `다음 질문` 버튼으로 다음 문제 이동
- 모든 질문 종료 후 완료 메시지 표시
- `localStorage`를 이용한 브라우저 내 결과 저장
- 반응형 레이아웃 지원

## 사용 기술

- HTML5
- CSS3
- JavaScript (Vanilla JS)
- Browser Local Storage

## 실행 방법

1. 프로젝트 파일을 같은 폴더에 둡니다.
2. `index.html` 파일을 브라우저에서 엽니다.
3. 원하는 선택지를 클릭해 결과를 확인합니다.

별도의 설치나 패키지 다운로드는 필요하지 않습니다.

## 동작 방식

### 1. 질문 출력
`app.js`에 저장된 질문 배열에서 현재 질문을 화면에 표시합니다.

### 2. 투표 처리
사용자가 A 또는 B를 선택하면 해당 질문의 `countA` 또는 `countB` 값이 증가합니다.

### 3. 결과 계산
전체 투표 수를 기준으로 각 선택지의 비율을 계산하고, 아래 형식으로 화면에 보여줍니다.

```text
선택지A 54% (12표) vs 선택지B 46% (10표)
```

### 4. 데이터 저장
투표 결과는 `localStorage`의 `balanceQuestions` 키에 저장됩니다.

즉,
- 새로고침 후에도 결과 유지 가능
- 같은 브라우저에서는 이전 투표 데이터 반영 가능
- 다른 기기나 다른 브라우저와는 데이터가 공유되지 않음

## 커스터마이징 방법

### 질문 추가
`app.js`의 `defaultQuestions` 배열에 아래 형식으로 객체를 추가하면 됩니다.

```javascript
{
    question: "평생 커피만 마시기 vs 평생 탄산만 마시기",
    optionA: "평생 커피만 마시기",
    optionB: "평생 탄산만 마시기",
    countA: 0,
    countB: 0
}
```

### 질문 수정
기존 배열 안의 `question`, `optionA`, `optionB` 값을 원하는 문장으로 바꾸면 됩니다.

### 스타일 수정
- 전체 배경: `body`
- 게임 카드 영역: `.container`
- 선택 버튼: `.option-btn`, `.option-a`, `.option-b`
- 결과 영역: `#result`
- 다음 버튼: `#next-btn`

## 로컬 저장 데이터 초기화

테스트 중 저장된 투표 결과를 초기화하고 싶다면 브라우저 개발자 도구 콘솔에 아래 코드를 입력하면 됩니다.

```javascript
localStorage.removeItem("balanceQuestions");
location.reload();
```

## 프로젝트 특징

이 프로젝트는 프론트엔드 기초 학습용으로 적합합니다.

특히 아래 내용을 연습하기 좋습니다.

- DOM 선택 및 이벤트 처리
- 배열 기반 데이터 렌더링
- 조건문과 상태 관리
- 브라우저 저장소 활용
- 반응형 UI 구성

## 참고 사항

현재 투표 결과는 서버 DB가 아니라 브라우저 `localStorage`에 저장됩니다.
따라서 실제 서비스처럼 모든 사용자의 결과가 합산되지는 않습니다.
전체 사용자 공통 데이터를 반영하려면 백엔드와 데이터베이스가 추가로 필요합니다.
