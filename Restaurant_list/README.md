# 나만의 맛집 리스트

간단한 **HTML, CSS, JavaScript 기반 맛집 리스트 웹앱**입니다.  
맛집 이름, 카테고리, 한 줄 평을 입력해 목록에 추가할 수 있고, 등록된 맛집 개수를 확인하거나 개별 항목을 삭제할 수 있습니다.

## 프로젝트 소개

이 프로젝트는 자주 가는 맛집이나 나중에 방문하고 싶은 식당을 직접 기록할 수 있도록 만든 웹앱입니다.  
복잡한 기능보다 **입력, 목록 렌더링, 삭제, 개수 표시** 같은 프론트엔드 기초 기능을 익히는 데 초점을 맞췄습니다.

## 주요 기능

- 맛집 이름, 카테고리, 한 줄 평 입력 후 목록에 추가
- 등록된 맛집 개수 실시간 표시
- 맛집 카드 형태로 목록 출력
- 항목별 삭제 기능 제공
- 목록이 비어 있을 때 안내 문구 표시
- 이름 입력창과 한 줄 평 입력창에서 **Enter 키로 추가 가능**

## 기술 스택

- **HTML**
- **CSS**
- **JavaScript (Vanilla JS)**

## 파일 구성

```bash
.
├── restaurant.html   # 메인 HTML 파일
├── style.css         # 전체 스타일 정의
└── app.js            # 맛집 추가/삭제 및 렌더링 로직
```

## 실행 방법

1. 프로젝트 파일을 같은 폴더에 저장합니다.
2. `restaurant.html` 파일을 브라우저에서 엽니다.
3. 맛집 이름, 카테고리, 한 줄 평을 입력한 뒤 **추가** 버튼을 누르면 목록에 등록됩니다.

## 동작 방식

### 1. 맛집 데이터 관리
- `restaurants` 배열에 맛집 정보를 객체 형태로 저장합니다.
- 각 객체는 `name`, `category`, `review` 값을 가집니다.

### 2. 목록 렌더링
- `renderList()` 함수가 현재 배열 상태를 기준으로 화면을 다시 그립니다.
- 등록된 맛집이 없으면 빈 상태 메시지를 보여줍니다.

### 3. 삭제 기능
- 각 카드의 삭제 버튼을 누르면 해당 항목을 배열에서 제거한 뒤 목록을 다시 렌더링합니다.

### 4. 입력값 검증
- 맛집 이름 또는 한 줄 평이 비어 있으면 `alert`로 안내하고 입력을 막습니다.

## 화면 구성

- 상단 제목
- 등록 개수 표시 영역
- 맛집 이름 / 카테고리 / 한 줄 평 입력 영역
- 맛집 카드 목록 영역

## 아쉬운 점 / 개선 아이디어

현재 버전은 학습용으로 적합한 구조이지만, 다음과 같은 기능을 추가하면 더 완성도 높은 프로젝트로 발전시킬 수 있습니다.

- **LocalStorage 연동**으로 새로고침 후에도 데이터 유지
- 카테고리별 **필터링 기능**
- 수정 기능 추가
- 평점 기능 추가
- 모바일 화면에 맞춘 반응형 UI 강화
- 유효성 검사 메시지를 `alert` 대신 화면에 직접 표시
